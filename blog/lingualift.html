LinguaLift ‚Äî a frictionless way to learn from your real reading

Published Oct 7, 2025 5 min read

A minimal workflow that lets you grab any phrase while browsing, translate it instantly, tag by context (work, travel, football, etc.), and review the best bits later. Built with a tiny Chrome helper and a Make scenario ‚Äî no heavyweight app needed.

Capture from anywhere

Highlight text in Chrome and send it with one click. The page title and URL are captured for context.

Translate & enrich

Make calls an LLM/translation step, adds phonetics (if needed), and suggests tags based on topic.

Review without friction

Entries land in a simple table (Notion/Sheets/Airtable‚Äîpick your store) with filters for quick sessions.

How it works

  1. Chrome helper: a minimal extension adds a context-menu entry ‚ÄúSend to LinguaLift‚Äù. It packages selection, source URL, and language.
  2. Make webhook: receives the payload, translates the selection, and proposes tags (domain + tone).
  3. Storage: the record is saved to your store (I use Notion DB; Google Sheets works too).
  4. Drill: a review view shows ‚Äúnew / due / starred‚Äù. Optional spaced-repetition field increments on each pass.

Set it up in 10 minutes

Use the files below, change the webhook URL in the extension, and pick your data store. The flow is provider-agnostic.

Minimal configuration

In extension/background.js, set your Make webhook:

const WEBHOOK_URL = "https://hook.eu1.make.com/your-webhook-id";
chrome.runtime.onMessage.addListener((payload) => {
  fetch(WEBHOOK_URL, { method: "POST", body: JSON.stringify(payload) });
});

In Make, map fields like text, source_url, lang_from, lang_to, then add:

  • Translate (provider of your choice)
  • Generate tags (LLM) ‚Äî limit to 2‚Äì3 concise tags
  • Write row to Notion/Sheets/Airtable
Privacy note: keep sensitive content out of the capture flow. For public pages, the URL is stored for context; disable this if you prefer local-only logs.
